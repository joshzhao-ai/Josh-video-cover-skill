# 封面生成 SOP（定死标准）—— 任意视频/图 → 高质量封面

> 配套《douyin_cover_standard.md》（官方5步合格标准）一起用。
> 核心信念：**任何输入，都先去优质案例库里找最接近的参考图来"锚定"，再适配主体。** 不凭空裸生成。

样板案例：麻醉医学科普 `anes_v2_1.png`（用「奥施康定」参考图 image2image 锚定 + 文字层铁律）。

---

## 流程（6 步，按顺序）

**① 看懂内容**：抽帧自己看 → 主体是什么、主题、钩子点、**有没有真人**。

**② 判垂类**：归入 6 大垂类之一 —— 人文社科 / 生活记录 / 美食 / 财经 / 科技 / 科普。

**②.5 人像确认环节（视频有真人出镜时必走）**：询问用户 3 选 1 —— (a) **上传本人照片**（推荐·最可控）；(b) 视频抽帧（实验·低清易致 identity 漂移，慎用）；(c) 不要人像/符号化。需要人像时，用即梦 image2image 把人像**融进**生成封面（5.x 实测可行）。
   - **具体打法（v3·垫图两步法，用户定调+R1验证）**：要让成品**真的像参考家族**，参考图必须垫进模型，但分两步走、各管一件事：
     - **Step1 垫底图**：image2image(**只喂参考封面**) → 提示词"严格保留版式骨架/配色/字体气质 + **内容全部替换**为本视频主题 + **原人物去掉、位置留空** + 标题换成我们的 + 删除原图一切其它文字和@水印 + 列出禁项（如不要K线/钞票/原主体）"。产出一张继承参考 DNA 的纯底图。
     - **Step2 融人**：image2image(**[人像, 底图]** 逗号分隔) → 提示词"人物用第一张照片这个人（长相不变）放进底图预留位 + **底图所有文字逐字保留、版式/道具一个不许丢** + 人重打光匹配底图（色温/方向/对比/边缘/景深）"。
     - 历史教训链：v1 整张参考+人像一把喂 → 内容泄漏（美元/机房残字）；v2 参考完全不进模型只描述骨架 → DNA 全丢、千篇一律；**v3 两步垫图 = 既继承又不泄漏**。
     - 已知风险（R1 实测）：Step2 可能改坏底图小字、顶掉底图道具 → fuse prompt 必须"逐字锁文字 + 道具与人共存"。
   - **模式三（v4·风格化重绘，用户即梦实测配方，见 style_repaint_recipe.md）**：插画/涂鸦/丝网印刷/复古海报类风格首选。**单步** image2image(只喂人像)，提示词="风格词(大师级排版/极繁主义/半调/杂色/丝网印刷/点线面层次) + 核心主题 + 视觉风格(人物用照片里的人、指定动作、风格不要变) + 信息细节清单"。人物**以海报风格重绘并参与画面(可换姿势)**——天然无贴图感、无层次混乱、文字一次成型(实测3条要点~40字全对)。
   - **模式选择**：摄影感真人版式参考(小Lin说类) → 两步垫图；插画/印刷海报风 → 风格化重绘；"底图道具+贴人"两层共存易乱(影视飓风版教训) → 改风格化重绘。
   - **🔧 已编码进 `pipeline/cover_pipeline.py`（人像门，用户2026-06-10定调）**：检测到 `has_real_person=true` 且未决策 → 流水线**自动停下主动问**"要不要把人像放进封面"，而非默默退无人。选「放」(`--person-mode uploaded-photo --portrait <图>`) → 风格化重绘；选「不放」(`--person-mode no-person`) → 无人方案(按垂类分流：产品影棚/医学科普/符号海报)。
   - **文字排版铁律（麻醉 symbolic_a + iPhone拆机 教训，已写进 TRIM_RULE）**：① **字号极大**——主标题横向占画面 75-90%，绝不小或纤细(否则像方案2看不清)；② **留安全边距**——距画面上/左/右边缘留空隙，绝不顶边或被裁(否则像方案1画面不对)；③ **撞色衬底**——标题铺一块与背景撞色的实色块/粗描边(蓝底配橙块这种)，任何背景都跳脱(方案3被选中的关键)；④ 避开辉光与复杂纹理。

**③ 参考图锚定（⚠️ 2026-06-10 更新为"可选增强"，不再强制）**：早期最高铁律曾要求"每张必须 image2image 垫参考图、绝不裸生成"，因为**当时**裸生成土/雷同/标题小。**现已更新**——即梦 5.0 + 强题材配方（food/医学/设计/产品/实物操作等）+ 排版铁律 TRIM_RULE（字号 75-90% / 撞色衬底 / 留安全边距）后，**无人路线 text2image 直生已稳定到 9.5、不再土、标题不再小**（6 条真实视频验证；用户 2026-06-10 拍板采用"配方直生"）。结论：**默认走配方直生**；垫图降级为**可选增强**（只在想严格继承某张参考的版式时才用）。**真人路线**仍 image2image 只垫【用户本人照片】。以下 ③④ 的垫图规矩，作为"确实选择垫图时"的操作参考保留——
在 `优质案例/{垂类}/竖封面` 里，按「主体类型 + 主题 + 画风」挑**最接近的 1～2 张**优质案例当锚。
- 科普"画风跟主题走"：医学→插画类参考(奥施康定)、地理/旅行→真实摄影参考、技能→图形参考…
- **铁律：参考图的主体/构图必须"对得上"新内容**（医疗找医学插画、单主体找产品版式如天玑芯片）。**拿不相干的硬套会出四不像**——本案例反例：拿财经《房产税》(城市)去锚麻醉，结果"看不懂"。对得上才锚得好（B 用奥施康定=好，A 用城市=垃圾）。
- **根治"标题太小"：专挑一张"标题大、风格对味"的参考来锚**（image2image 会继承参考的标题比例）。text2image 裸生成几乎必然标题偏小。本案例 A 用科技《影视飓风》大标题封面锚定后，标题立刻够大够酷。
  - **（2026-06-10 更新）** 此"裸生成标题必小"为旧结论；现已由 TRIM_RULE 排版铁律（强制标题占画面 75-90% + 撞色衬底 + 留安全边距）解决，text2image 直生的标题也够大够清晰（iPhone拆机 / 日料组验证）。

**④ 风格锚定生成**：
- 库里有接近的参考 → **即梦 image2image**：参考图当底子（锚定画风/配色/质感/排版气质），提示词把**主体替换**为新视频内容，并写明"保留参考风格、不要参考图里的原主体元素"。
- 库里没有接近的 → 退回 text2image 自由发挥（按该垂类规则）。
- 模型：`dreamina ... --model_version=5.0 --ratio=3:4 --resolution_type=2k`

**⑤ 文字层铁律**（本案例定死，所有垂类通用）：
1. 标题清晰**严禁用生硬黑色矩形色块/黑条**（会与画面断层、显土）。正确做法：①标题压在**画面本身较暗的区域** + ②白字加**细微深色描边 + 柔和投影**；如需压暗只用**极柔和、看不出边界的大范围渐变**，与画面融为一体。标题**两行水平居中对齐**。
   - **水印=绝对红线**：image2image 必须在提示词明确"**去掉参考图自带的创作者水印/署名**"（极易被一起复制过来，如 @极客湾/@迷因水母）；**左下角或任何位置出现水印 → 判官直接打回，绝不交付**。
2. 标题 **≤10 字 / 单行≤5 字**，疑问句或关键词钩子，字体**硬朗有设计感**，**关键词用亮色高亮**。
   - **标题必须留足顶部/左右安全边距，绝不贴边、不被画面裁切**（官方红线#3 完整性；本次 final_A 顶行被切即反例）。
3. 加**小排版点缀**提升丰富度：角标/标签（如「医学科普」「完整版」）、底部一行装饰英文小字。
4. 长英文/拉丁词只当**装饰小字**，容忍偶发拼写瑕疵；**关键信息只用中文**（即梦中文准、长英文易错）。
5. **不编造真人脸**（官方红线）：真人题材 → 口罩遮挡 / 扁平插画 / 符号化，绝不生成可辨识假人脸。

**⑥ 多版挑最佳 + 判官验收**：每次出 **≥3 版**挑最佳；用《官方5步标准》逐项判（底图清晰明亮 / 主体完整 / 文案凝练 / 主次有焦点 / 色彩和谐对比）。**任一红线命中 → 打回重生**。

**交付标准（用户实测定调）**：给用户的 3 张里，**至少 1 张达到"优秀/可当主推"水平**，其余可用且与主推差异化即可；**任何硬伤一律不交付**。
**优秀最可靠的来源 = 参考图匹配得越准越好**（本案例 B 优秀，正因奥施康定与医学主题完美匹配；A 难，因库里无"医疗真实摄影"完美参考）。→ 实践启示：**优先挑库里匹配最准的那张做主推方向**。
