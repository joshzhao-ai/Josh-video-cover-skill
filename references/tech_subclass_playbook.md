## 二、三个 case 暴露的共性问题（科技垂类通病）

1. **主体泛化 —— 画"一类东西"而不是"这一个东西"**（最致命）。
   天玑被画成"芯片们"而非"天玑9400 这颗芯片"，还混入竞品 logo。根因是 skill 只知道"这是数码产品测评"，不知道"主体实体 = MediaTek 天玑9400 芯片"。
2. **标题太克制 —— 白字 PPT 感**。官方科技封面标题几乎都是高饱和大字（金/橙），冲击力是点击理由的一部分；skill 默认白字描边，干净但弱。
3. **构图太居中太整齐 —— 缺"信息流取舍"**。skill 爱对称居中、整齐阵列；官方是"前景主推 + 背景铺陈"，杂而有重点，更像真人设计师的取舍。
4. **丢真实封面语境** —— 缺创作者水印、缺内容密度（结构力学箭头偏少）、缺选购对比语境。
5. **太干净 = 假**。悬浮发光卡片、官网素材式整齐，是"AI 规整输出"的指纹。

> 一句话：**skill 现在擅长生成"清楚的图"，还不擅长生成"有具体主体 + 创作者取舍 + 真实语境"的封面。**

## 三、三个子类的可复用生成方法

### 子类 A — 数码芯片 / 单品解析

- **适合视频**：单一产品/芯片的评测、技术解析、首发实测（天玑9400、骁龙8 Gen4、iPhone16 Pro 评测、某显卡测评）。
- **主体怎么选**：**标题指向的那一个具体型号产品本身**，单主体、居中偏下、大（占 40-55%）。芯片要带正确品牌标识（天玑→MediaTek logo），手机/平板要是那一款的真实形态。
- **标题怎么排**：产品名超大（金白渐变 / 银色金属质感），副标题（技术解析/评测/首发）次行小一号。标题压顶、有冲击。
- **背景怎么处理**：纯黑摄影棚，产品局部冷光/反光，轻微悬浮投影。
- **视觉风格**：真实产品渲染图 / 发布会主视觉质感。
- **必须保留的官方策略**：① 单一具体产品主体；② 品牌标识正确；③ 大字冲击。
- **禁止泛化元素**：🚫 一排泛芯片/泛设备；🚫 **竞品或无关品牌 logo（Apple/骁龙/英伟达等，除非就是该品牌）**；🚫 悬浮发光卡片陈列；🚫 cyberpunk/AI 节点/K线。
- **Prompt 怎么写**：
  > 主体必须是「{具体型号，如 MediaTek 天玑9400}」这一颗/台产品本身的高质感 3D 棚拍特写，带正确品牌标识；单主体居中偏下占画面 45%；纯黑摄影棚 + 冷光反光 + 悬浮投影；像产品发布会主视觉。**绝对禁止：画成一排/多颗泛芯片或泛设备；出现 Apple/骁龙/英伟达等无关或竞品 logo；悬浮发光卡片；cyberpunk/AI 节点/K线。** 标题金白渐变超大字压顶。

### 子类 B — 选购指南 / 推荐避坑

- **适合视频**：X 怎么选 / 选购指南 / 推荐 / 避坑 / 全档位对比（iPad 选购、千元机推荐、显示器选购）。
- **主体怎么选**：**多款不同型号并置**（体现"全档位/多选择"，8-12 台不同型号铺陈）+ **前景一台主推特写**。两层结构，不是单层整齐阵列。
- **标题怎么排**：**高饱和强冲击大字（橙 / 黄）**，面积大、压迫感强——指南类标题不能克制。
- **背景怎么处理**：暗底 + 多机型铺陈 + 动势线条（紫光/流线）增强信息流冲击。
- **视觉风格**：信息流爆款封面感——**杂而有重点**，不是整齐官网陈列。
- **必须保留的官方策略**：① 多型号对比语境（"这么多怎么选"）；② 橙/黄强标题；③ 前景主推 + 背景铺陈两层。
- **禁止泛化元素**：🚫 少量整齐陈列（丢"多选择"语境）；🚫 克制白字；🚫 单一产品；🚫 全部同款只是尺寸不同。
- **Prompt 怎么写**：
  > 背景铺陈大量不同型号的{产品类，如 iPad}（不同尺寸/颜色/年代，体现全档位），前景一台主推产品特写；暗底 + 紫色光线动势；表达"这么多怎么选"的选购语境。标题用**橙色或黄色高饱和超大粗字**，面积大、冲击强。禁止：只放几台整齐相同产品；克制白字；纯官网素材拼贴感。

### 子类 C — 工程教学 / 图示科普

- **适合视频**：学科课程、工程示意、教材科普（结构力学、电路、机械原理、微积分图解）。
- **主体怎么选**：真实教学图示（桁架/电路/分子/几何），**标注要密集专业**（红蓝箭头多 = "内容真的被讲解过"）。
- **标题怎么排**：4 层文字系统（kicker「这样的」+ 立体拼装大字 + 白色疑问斜标签 + 橙色系列标签）。
- **背景怎么处理**：**深色纯背景、偏暗有层次**（不要偏亮，亮会降专业感）。
- **视觉风格**：教材图示 + 强标题系统；**禁止泛科技 UI / 数据面板 / 赛博光效**。
- **必须保留的官方策略**：① 教学图示的专业标注密度；② 4 层标题系统；③ 深暗背景；④ 底部作者水印。
- **禁止泛化元素**：🚫 泛科技 UI / 赛博霓虹；🚫 过简示意图（标注太少）；🚫 过亮背景。
- **Prompt 怎么写**（已基本做到，微调）：在现有 academic 路由基础上：箭头标注密度提高（贴近官方"密集标注"）、背景压暗一档、补底部作者水印位。

## 四、写回 Skill 的分层映射

| skill 层 | 要加/改什么 | 解决哪个共性问题 |
|---|---|---|
| **analysis** | ① 科技垂类细分出 `tech_subclass ∈ {chip-review, buying-guide, edu-diagram}`；② **新增 `subject_entity` 字段**——VLM 提取标题指向的"具体实体"（如"MediaTek 天玑9400 芯片"），而不只是泛 key_elements | 🔴 主体泛化（根因）|
| **title / subtitle 处理** | 按子类定标题策略：芯片=产品名金白大字、选购=橙黄强冲击大字、教学=4 层系统 | 🟠 标题太克制 |
| **prompt_planner / style routing** | `is_digital_review_source` 拆成 `is_chip_review` / `is_buying_guide`（`is_academic_engineering` 已覆盖 edu-diagram）；按子类路由到不同 subject/composition/palette/visual_system | 子类风格走对 |
| **prompt template** | 每子类的 subject 锁定 `subject_entity`、禁止竞品 logo（芯片）、多档对比（选购）、密集标注（教学）；composition 引入"前景主推+背景铺陈"（选购）；palette 强标题色（选购橙黄） | 🔴 主体/🟠 构图/🟠 语境 |
| **（已完成）judge** | 对标官方 `subject_accuracy` + `reference_inheritance` 维度——作为验证工具，不再扩展 | 评测有效性（已修复）|

**根因修复的灵魂**：在 **analysis 层产出 `subject_entity`**，让 prompt 围绕"天玑9400 这一颗芯片"而不是"数码产品"这个类生成。这是从"画一类东西"到"画这一个东西"的根本转变，是天玑/iPad 主体问题的总开关。
