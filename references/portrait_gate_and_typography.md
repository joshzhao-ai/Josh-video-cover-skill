# 人像门终版(3选项) + 取帧抠图转正 + 字体排版升级（2026-06-10 用户拍板）

## 一、人像门 3 选项（真人口播类必触发）
`subject_strategy == real_person_talking_head` 且 `has_real_person` → pipeline 自动停下问用户，3 选 1：
1. **uploaded-photo**：上传本人照片 → 风格化重绘。**效果最佳**(Codex/Claude Code 那组 showcase)。产品上应优先引导走这条。
2. **frame-cutout**：从视频取一帧清晰正脸 → 风格化重绘。**次优**，无需额外上传；实测复古相机测评的"涂鸦/动感"变体用户判定"特别好"。注意：人物是**风格化重绘(神似非精确)**，样貌会变。
3. **no-person**：不放人像 → 按 `no_person_fallback` 字段走无人配方(mood/product/food/hands/symbolic)。
**非口播类**(产品/实物操作/美食/氛围)即使有真人氛围镜头也**不触发**(封面不以口播者为主角)。

## 二、取帧抠图"转正"（推翻旧"video-frame 慎用"）
旧 SOP 说视频帧低清易 identity 漂移、慎用——那针对的是"**抠图贴图**"(会漂会假)。而"**取帧 → 风格化重绘**"把人物按海报风重绘、避开了贴图感，实测效果好。故取帧作为"用户没上传照片时的次优方案"**转正**。优先级：**上传照片 > 取帧重绘 > 无人**。

## 三、字体排版升级（纠偏"色块滥用"）
撞色块衬底当初为救"标题看不清"而加，但被滥用成每张"整条纯色块 + 平铺字"，单调、设计感弱(用户点名批评)。TRIM_RULE 已改：**字体设计多样化**(艺术字/书法/复古印刷字/立体描边+材质/关键词局部异色/与画面穿插)，**色块只作局部点缀、不铺满整行**。正例：日料毛笔字、封面设计描边艺术字、Codex 复古印刷标题。

## 四、环绕元素跟主题（pick_motif）
风格化重绘的环绕涂鸦元素随主题变：编程→代码/终端；相机→相机/胶片/光圈；数码→设备图标。不再写死"代码符号"。

## 五、评分诚实（纠"虚高"）
天玑 v2(黄底白字/构图平)被用户判**不及格**。判分对标 **美感/构图/冲击力**，不是只看"文字清晰"。不再自我感觉良好地给 9 分。
