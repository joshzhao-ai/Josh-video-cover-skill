# 学科教育 / 工程教学 垂类规则

> 别名："奥达升 AWE 风格"、"极简主义工程科普海报"
> 参考基准：奥达升 AWE《结构力学 / 桁架内力篇》系列封面
> 反向参考：iter-1 教学元素堆砌、AI 工具 four-layer-background 模板

这份文档是 `scripts/build_cover_prompts.py` 里 `is_academic_engineering_source()` 路由背后的设计依据。改 skill 时回来读这份文档，不要让任何改动破坏这套垂类范式。

## 1. 触发条件

`is_academic_engineering_source(source)` 命中以下任一即视为本垂类：

**强词（任一命中即匹配）：**
- 力学类：结构力学 / 材料力学 / 理论力学 / 弹性力学 / 流体力学 / 土力学 / 岩土力学
- 工程构件：桁架 / 应力 / 弯矩 / 剪力 / 内力 / 腹杆 / 弦杆 / 拉杆 / 压杆
- 电学：电路 / 模拟电路 / 数字电路 / 电磁场 / 电动力学
- 机械：机械原理 / 机械设计 / 机械制造 / 齿轮传动
- 数学：微积分 / 线性代数 / 概率论 / 数学分析 / 高等数学
- 物理：量子力学 / 热力学 / 统计物理 / 凝聚态
- 化学：有机化学 / 物理化学 / 无机化学 / 分析化学
- 生物：分子生物学 / 细胞生物学 / 遗传学 / 生物化学
- 教育阶段：考研 / 网课 / 公开课

**弱组合（教育平台词 + 学科领域词同时命中）：**
- 教育平台：湖南大学 / 清华大学 / 北京大学 / 教授 / 教材 / 课件 / 课程 / 讲解
- 学科领域：力学 / 工程 / 数学 / 物理 / 化学 / 生物 / 电学 / 材料 / 土木 / 建筑

误命中代价（教材风出现在错误视频上）远小于错配到 cyberpunk AI 工具模板的代价，所以判定阈值偏宽松。

## 2. 视觉系统：极简主义工程科普海报

### 2.1 整体范式

- **画面结构**：上 40% 文字层级 + 下 60% 工程结构主体，中间清晰留白呼吸空间
- **核心气质**：Bauhaus 工程图纸 + 现代信息设计 + 海报感
- **绝对禁止**：four-layer-background AI 工具语法（底层网格 + 中层模糊 UI + 上层粒子光效）
- **关键原则**：**宁可元素少、留白多，也不要把画面塞满**——这是和 iter-1 教学堆砌路径的本质区别

### 2.2 调色板

```
背景：学院深蓝 #0E1A33 ~ #1A2A4D（不要网格暗纹、不要梯度光斑）
主标题文字：纯白色 + 银色金属螺丝铆钉细节
副标题标签：
  - 暖橙色块 #E6661C ~ #E6741C + 黑色粗体大字（不要偏黄橙、不要荧光橙）
  - 纯白色块 + 黑色无衬线粗体字（"你喜欢吗？" 类疑问标签）
工程结构线条：纯白色或极浅米白色（极细线宽，CAD 工业制图感）
功能色箭头：
  - 红色：受压 / 压杆 / 风险 / 关键节点
  - 蓝色：受拉 / 向量 / 流动 / 辅助说明
```

### 2.3 4 层文字结构（这是这个垂类的灵魂）

| # | 位置 | 内容 | 样式 |
|---|---|---|---|
| ① | 主标题**左上角外侧**（不压字、有间距）| `{kicker}`（默认"这样的"）+ 像素鼠标 icon | 高明度白色无衬线粗体，垂直 5-7%；鼠标光标是像素化锯齿边缘（非矢量平滑）|
| ② | 顶部居中 | `{main_title}`（例如"结构力学"）| **白色立体厚切拼接风格**；字宽字距严格均等像系统粗黑体；每个字 3-4 颗银色金属螺丝铆钉**对称分布在四角**；垂直 22-30%，水平 65-85% |
| ③ | 主标题右下/右侧（与 ④ 错位）| `{question}`（默认"你喜欢吗？"）| 纯白色矩形 + 黑色无衬线粗体；**整体向右倾斜 -6°~-10° 悬浮斜放**；高约主标题字高的 30-40% |
| ④ | 主标题下方居中偏左 | `{bottom_label}`（例如"桁架内力篇"）| **暖橙色 #E6661C-#E6741C** 矩形 + 黑色粗体大字；**轻微 -3°~-6° 微斜**；宽度严格 55-70%（不与 ③ 重叠）|

**critical：③ 和 ④ 必须在水平方向错位**——左下橙色 + 右上白色的斜接构图，绝对不能重叠（这是 iter-4 batch 1 发现的硬性约束）。

### 2.4 工程结构主体规则

- **扁平 2D 工业线框图**（critical：不是 3D 立体建模、不是带轮廓光的 3D 渲染、不是等距视图）
- 用纯白色 / 极浅米白色细线条勾勒（线宽均匀，像 CAD / Adobe Illustrator 工业制图）
- 杆件交叉处用小圆点标记节点
- 每个结构两端必须有**锚定三角支座**——倒立小三角形 + 下面 3-5 条短斜线（虚线阴影）表示固定端
- **几何严谨性**：每个结构左右轴对称、杆件长度按工程比例统一、节点位置精确等距，像 AutoCAD 工程模板出来的，不要手绘感、不要节点错位
- 垂直堆叠 2-3 个结构，彼此宽度大致一致（占画面水平 70-85%），高度有规律变化
- **底部留白**：最下方那个结构距画面底边留 5-10% 高度的纯背景呼吸空间

### 2.5 红蓝箭头规则（critical 节制 + 统一）

- 每个结构上**只叠加 4-8 个箭头**——绝对不要每根杆件都画箭头、不要密集铺满
- **箭头统一性**：所有箭头必须用同一套 vector 风格——同样的箭杆粗细、同样的箭头三角形头部、同样的长度比例，像 Illustrator 一个 symbol 复用
- 箭头长度约为相邻杆件长度的 30-50%
- 方向清晰锐利，红色 = 受压、蓝色 = 受拉 / 向量

## 3. Hook 偏好（标题选择）

学科教育的真实情感驱动是"学懂了 / 这样讲我能听明白 / 学科本身有趣"，**不是** AI 工具圈的 risk / reversal / result / pain。

| Hook 类型 | 模板 | 示例 |
|---|---|---|
| 疑问 + 情感共鸣 | 「这样的 {学科} 你喜欢吗？」「学过 {学科} 的都笑了」| 这样的结构力学你喜欢吗？|
| 共鸣型 | 「{考研党 / 土木学生 / 物理生} {狂喜 / 收藏 / 必看}」| 考研党狂喜：1 小时讲完概率论 |
| 认知差 | 「3 分钟讲透 {主题}」「{主题} 的真相」「学校不教的 {主题}」| 学校不教的微积分起源 |
| 学科趣味 | 「原来 {学科} 还能这样讲」| 原来量子力学还能这样讲 |

副标题**默认作系列分类标签**：「{topic}篇」（如「桁架内力篇」「应力篇」「内力篇」「腔体篇」）。

## 4. 硬禁止清单

绝对不允许出现在学科教育封面里：

- 代码终端 / IDE 窗口 / 命令行光标
- 浏览器 / 网页 UI 截图
- K 线 / 股票走势 / 纳斯达克曲线 / 财务报表
- AI 节点关系网络 / 神经网络节点图
- 加密货币 / 区块链元素
- Cyberpunk 霓虹发光 / 青色科技体积光
- 漂浮粒子辉光 / 发光全息卡片 / 科幻 HUD

**教学装饰元素**（让画面变密、丢失极简感）：
- 每个结构下面的 caption 文字小字
- 🏠/🏛/🌉 等应用场景 emoji 或 icon
- 公式注解小字（如"N=弦杆内力"）
- bullet 点状标签
- 底部 bilibili / 校徽水印
- 教材章节小字 / 目录号 / 序号 / 坐标轴标注

除主标题、kicker 小字、彩色矩形标签上的文字外，画面里不要出现任何其他可读文字。

## 5. 生僻字与采样策略

### 5.1 中文生图模型的生僻字问题

Seedream 等中文 image-gen 模型对低频汉字（「桁」「弦」「矩」「杆」等专业术语）鲁棒性差，**经常错写成形近字**（「桁」→「析」/「梁」/「橫」/「樯」）。

**对策**（按优先级）：

1. **多次采样取最佳**：同一 prompt 跑 3-5 次，Seedream 会有一次掷骰子掷出正确字形的，这是最稳的方式（iter-4 b4 就是这样选出的）
2. **Prompt 字形提示**：在 `build_cover_prompts.py` 的 `RARE_CHAR_HINTS` 字典里登记字形描述（如"桁字 = 木字旁 + 行字"），自动追加到 prompt 末尾。**实测对 Seedream 帮助有限**（模型看到"不要写成析"反而强化了"析"作为候选），但成本几乎为零，保留作为防御性指令
3. **PIL 局部修字**（`scripts/fix_rare_char.py`）：检测橙色矩形位置，采样真实橙色色值，用系统中文字体覆盖正确字。**仅当原图确实有错字时才用**——盲修会让正确写出来的字被多盖一层

### 5.2 采样策略推荐

```bash
# 同一 prompt 跑 3-5 批，从所有 info-heavy 候选里人工挑选最完美的一张
for b in b1 b2 b3 b4 b5; do
  python3 scripts/generate_ai_covers.py \
    --prompts prompts.json \
    --output-dir covers_$b/
done
# 然后人眼对比 5 张 info-heavy.jpg，选最好那张
```

每张图 1-2 元，5 批约 5-10 元的成本就能拿到 9.5+/10 的封面。**这种采样成本对工程美学要求高的封面是合理代价**。

未来如果要做成 skill 自动行为，应该在 `run_cover_workflow.py` 加 `--auto-pick-best N`，多批采样后用 VLM 评分挑选。

## 6. 验收 checklist（人工 review 时用）

| # | 检查项 | 通过条件 |
|---|---|---|
| 1 | 字体均衡 | 4 个汉字宽度高度严格等比、字间距均等 |
| 2 | 螺丝铆钉对称 | 每字 3-4 颗，分布在四角（左上、右上、左下、右下） |
| 3 | 桁架扁平 2D | 不是 3D 立体、不是带轮廓光 |
| 4 | 桁架几何对称 | 每个结构左右轴对称、节点等距、CAD 制图感 |
| 5 | 锚定三角支座 | 每结构两端都有倒三角 + 斜线阴影 |
| 6 | 节点圆点 | 杆件交叉处有小白圆点 |
| 7 | 红蓝箭头节制 | 每结构 4-8 个，不铺满 |
| 8 | 红蓝箭头统一 | 同样箭杆粗细、同样三角头部、等大 |
| 9 | Kicker 不压字 | 「这样的」在主标题左上外侧，有间距 |
| 10 | 鼠标 icon 像素感 | 不是矢量平滑、有锯齿边缘 |
| 11 | 白色斜放 | 「你喜欢吗？」-6°~-10° 悬浮 |
| 12 | 橙色暖橙 | #E6661C-#E6741C 范围、不偏黄橙 |
| 13 | 白橙错位 | 不在水平方向重叠 |
| 14 | 底部留白 | 第 3 结构距底边 5-10% |
| 15 | 文字正确性 | 「桁架」等生僻字写对 |
| 16 | 无装饰元素 | 没有 caption、icon、emoji、公式注解 |

iter-4 batch 4 raw 在这 16 项上有 14 项满分 + 2 项接近满分（箭头节制 8/10、橙色暖度 8/10）。
